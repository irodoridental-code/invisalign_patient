# InvisAlign Pro - セットアップ手順書

複数iPad・スタッフでリアルタイム共有できるウェブアプリの公開手順です。
すべて無料で完結します。所要時間：約30〜45分。

---

## 必要なアカウント（すべて無料）

- GitHub: https://github.com
- Supabase: https://supabase.com
- Vercel: https://vercel.com

---

## STEP 1：GitHubにコードをアップロード

1. https://github.com にアクセスしてアカウント作成（またはログイン）
2. 右上の「＋」→「New repository」をクリック
3. Repository name: `invisalign-pro`
4. Private（非公開）を選択 → 「Create repository」
5. 「uploading an existing file」リンクをクリック
6. このフォルダの中身をすべてドラッグ＆ドロップ
7. 「Commit changes」をクリック

---

## STEP 2：Supabase プロジェクト作成

1. https://supabase.com にアクセス → 「Start your project」
2. GitHubでサインイン
3. 「New project」をクリック
   - Name: `invisalign-pro`
   - Database Password: 任意のパスワード（保存しておく）
   - Region: `Northeast Asia (Tokyo)` を選択
4. 「Create new project」→ 2〜3分待つ

---

## STEP 3：データベースのテーブルを作成

1. Supabase ダッシュボードの左メニュー「SQL Editor」をクリック
2. 「New query」をクリック
3. `supabase_schema.sql` の内容をすべてコピーして貼り付け
4. 「Run」（または Ctrl+Enter）をクリック
5. 「Success」と表示されればOK

---

## STEP 4：Supabase の接続情報を確認

1. 左メニュー「Settings」→「API」をクリック
2. 以下の2つをメモしておく：
   - **Project URL**（例: `https://xxxxxxxxxxxxxxxx.supabase.co`）
   - **anon / public** キー（`eyJ...` で始まる長い文字列）

---

## STEP 5：Vercel にデプロイ

1. https://vercel.com にアクセス → 「Start Deploying」
2. GitHubでサインイン → 「Continue with GitHub」
3. 「Import Git Repository」→ `invisalign-pro` を選択 → 「Import」
4. Framework Preset: `Vite` を選択
5. 「Environment Variables」を展開して以下を追加：
   ```
   VITE_SUPABASE_URL    = （STEP4でメモしたProject URL）
   VITE_SUPABASE_ANON_KEY = （STEP4でメモしたanon key）
   ```
6. 「Deploy」をクリック → 2〜3分待つ
7. 「Congratulations!」が出たらデプロイ完了！

---

## STEP 6：URLをスタッフで共有

1. Vercel ダッシュボードに表示されているURL（例：`https://invisalign-pro-xxxx.vercel.app`）をコピー
2. スタッフのiPad・PCのブラウザでそのURLを開く
3. iPadのホーム画面に追加するとアプリのように使えます
   - Safari で開く → 共有ボタン →「ホーム画面に追加」

---

## コード修正時の反映方法

1. GitHubでファイルを編集（または新しいファイルをアップロード）
2. Commitすると自動的にVercelがビルド開始
3. 約1〜2分で全端末に反映される

---

## トラブルシューティング

### データが表示されない
→ Supabase の SQL が正しく実行されているか確認
→ Vercel の Environment Variables が正しく設定されているか確認

### リアルタイム同期されない
→ Supabase ダッシュボード → Database → Replication → supabase_realtime の Publication に patients / doctors / visit_dates が含まれているか確認

### その他
→ Vercel ダッシュボードの「Deployments」→「Functions」でエラーログを確認

---

## セキュリティについて

現在の設定はURLを知っている人なら誰でもアクセス・編集できます。
パスワード保護が必要な場合は Supabase の Authentication 機能を追加することをお勧めします（別途対応可能です）。
